# Suspicious Login Investigation

## Incident Overview

A security alert was generated by the identity monitoring system indicating a suspicious login attempt for a corporate user account.

The login activity appeared abnormal due to an unusual geographic location and login pattern inconsistent with the user's normal behavior.

The SOC team initiated an investigation to determine whether the login attempt represented legitimate user activity or a potential account compromise.

---

## Alert Triage

During the initial triage process, the SOC analyst reviewed the authentication event details.

Key triage questions:

- Does the login location match the user's typical geographic region?
- Was the login successful?
- Has the user recently traveled?
- Were multiple login attempts observed?

Initial analysis showed that the login originated from an unusual geographic location not previously associated with the user.

---

## Investigation Steps

1. Review authentication logs from the identity provider.
2. Identify the source IP address and geolocation.
3. Compare the login location with the user's normal login behavior.
4. Check whether the login occurred shortly after another login from a different country.
5. Verify whether multi-factor authentication was used.

---

## Log Analysis

Authentication logs revealed the following login activity:

| Timestamp | Username | Source IP | Location | Event |
|----------|----------|----------|----------|------|
| 09:02 | j.smith | 192.168.1.25 | Germany | Successful Login |
| 09:47 | j.smith | 185.203.116.42 | Russia | Successful Login |

Observations:

- The second login originated from a foreign country.
- The time difference between logins suggests possible impossible travel.
- The source IP address had not been previously associated with the user.

## Incident Classification

The activity was classified as a **suspicious login event potentially indicating account compromise**.

Although the login was successful, the geographic anomaly and login pattern raised concerns.

---

## Severity Assessment

Severity: Medium

Reason:

Login detected from an unusual geographic location inconsistent with the user's normal behavior.

---

## Response Actions

The SOC team recommended the following actions:

- Contact the user to verify whether the login was legitimate.
- Reset the user account password.
- Revoke active authentication sessions.
- Enable or enforce multi-factor authentication.
- Monitor the account for additional suspicious activity.

---

## Lessons Learned

This investigation highlights the importance of monitoring authentication activity and identifying abnormal login behavior.

Key takeaways:

- Identity monitoring alerts can help detect account compromise.
- Geographic anomaly detection is a useful indicator of suspicious activity.

---

## Analyst Notes

Further monitoring of the affected account is recommended to ensure no additional suspicious activity occurs.
